## What you'll learn (from post's "Tips for Getting Started")

Paul lists four tips near the end. The live bullets expand slightly with context
from the body of the post.

Live bullets (5):
1. Start with a clear subscription choice: Pro ($20/month) for light use, Max ($100/month) for daily coding, since the context window and rate limits determine how far each session goes
2. Set up a proper terminal (Ghostty or iTerm2, Zellij for multiplexing, Oh My Zsh) so you can run Claude Code alongside your editor and shell
3. Understand the context window: Claude holds roughly 200,000 tokens, auto-compacts when full, and sessions work best within 5 to 10 turns before starting fresh
4. Be specific in your prompts and correct early when Claude drifts, rather than letting mistakes compound across turns
5. Trust but verify: let Claude write the code, but always check research-critical output yourself
