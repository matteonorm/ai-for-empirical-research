AI tools make it cheap to produce code, but verifying that code is correct remains expensive. This video tackles the "verification debt" problem head-on, using an IPO event study (built from Jay Ritter's data and CRSP returns) as a worked example. It walks through git fundamentals for research (commits as audit trail, branches for experiments), code review practices, routing feedback through GitHub issues, and wiring results directly into LaTeX drafts via Overleaf so no number is ever hand-typed.
